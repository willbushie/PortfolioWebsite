Bin Packing Algorithm

Similar to the weight packing problem, this assigned project focused on evaluating time and space complexity. Instead of evaluating package weights, this project focused on evaluating 3D space. This project was a challenge, but was a very fun problem to solve.

20 Hours

Tools Used

What I Learned

This was my first time programming and working in a three dimensional space. I learned the usefulness of a coordinate system and how that can improve my understanding of the problem while developing.

Using my knowledge of OOP (Object Oriented Programming), I created classes for the smaller boxes and the larger bins. Utilizing these classes helped me better understand how to place the smaller boxes within the larger bins.

Visualization is a powerful tool, both for the user and the developer. Using Matplotlib to visualize the program output gave me a better understanding of what was happening than a two dimensional or terminal output would have.

Issues & Resolutions

What kind of algorithm should I use?

I decided that best fit could take far too long with larger inputs, and the opposite of best fit is inefficient. I landed on a first fit algorithm, that would evaluate subsections of the larger bin to place the current smaller bin into.

How to evaluate the three dimensional space?

Working in three dimensions can be difficult. I settled on a coordinate system which would start at one corner of the object (0, 0, 0) at the bottom left corner for example. Then, using the dimensions of the bins, I would be able to calculate position as well as determining orientation.

How will "used" space be noted?

All boxes are object, and if a smaller box is placed inside of a larger bin, it's attributes will be updated to denote that. This includes its position inside of the larger bin, and the larger bin will be updated to show that it contains the smaller bin.

How will the output be visualized?

The project did not require a visual output, so I could have settled with a two dimensional or simple terminal output. However, I thought it would be a better idea to integrate Matplotlib functionality to see what the filled bins would look like.

Matplotlib Issue

What I Would Do Differntly

I would add a method of re-evaluating the subsections within the larger bins. This could help in the end and ensure more volume is utilized well. This may increase the execution time of the program and result in the same problem initially put forth, but it would be worth researching further.

Instead of a single color for the used volume inside of a larger bin, I would distinctly color each kind of smaller bin. This would help distinquish which small bins were placed where.

In addition to modifying the colors of the utilized space, I would also output how many smaller boxes are contained within the larger bins. If the colors were changed and the output was given, I think it would drastically help see and understand where each smaller box was in the larger bins.
